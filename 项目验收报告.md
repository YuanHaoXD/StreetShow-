## 摘要

随着服饰电商的快速发展，商家对高质量模特试衣图的需求与日俱增，而传统拍摄流程成本高昂、周期漫长。本项目以"AI驱动的虚拟试衣"为核心命题，研发了 **StreetShow** 系统——一个面向街拍风格的模特试衣图智能生成平台。

系统以 **风格DNA驱动** 为核心差异化设计：通过多模态大语言模型（Qwen VL）分析上传衣物的风格基因（街头感、学院风、商务正式等），动态生成与服装气质高度匹配的场景、动作和光影方案，驱动 Gemini 图像生成模型输出专业级试衣效果图。区别于现有工具的简单"换衣"功能，StreetShow 能理解服装的风格语言，为每件衣服生成与其气质完全匹配的展示方案。

系统实现了四种核心功能模式：Lookbook 多场景风格展示、Pose Lab 服装驱动姿势推荐、Multi-Fit 多物件同时上身，以及标准单张试衣。结合 FastAPI 异步后端与 Next.js 前端，系统具备完整的云端 API 调用链路、结构化建议输出和历史记录管理能力。

**关键词：** 虚拟试衣；风格DNA；多模态大语言模型；图像生成；服饰电商

## 一、研究背景与目的

### 1.1 问题背景

在服饰电商行业，模特试衣图是商品展示和营销的重要组成部分。然而，传统的模特拍摄过程耗时且成本高昂，包括服装采购、模特聘请、摄影布景、后期修图等多个环节，给商家带来较大的经济与时间压力。

据统计，一套完整的商业模特拍摄流程平均耗费数千至数万元，且每次上新都需重复这一流程。对于中小商家而言，高昂的拍摄成本严重制约了产品上架速度和展示多样性。

### 1.2 研究目的

本项目旨在利用人工智能生成相关技术，构建一套能够结合模特服装版型、服装风格和模特形体特点等因素，生成符合服装和人物特点的街景模特试衣图生成系统，以：

- **降低商家对真实模特拍摄的依赖**，减少拍摄成本
- **提高服装展示的灵活性**，支持多场景、多姿态展示
- **提升试衣体验的专业性**，超越简单换衣，实现风格化展示
- **为消费者提供个性化试衣参考**，辅助购买决策

### 1.3 项目定位

StreetShow 的核心定位是：**不只是"换上衣服看效果"，而是理解衣服的风格语言，为每件衣服生成与其气质完全匹配的展示方案。**

## 二、国内外研究现状

### 2.1 虚拟试衣技术发展

虚拟试衣技术经历了从二维图像合成到三维建模，再到基于深度学习和扩散模型的演进：

**早期阶段（2018年前后）：** 以二维图像合成为主，依赖简单的图像拼接技术。VITON（2018）提出基于人体姿态迁移的试衣方法，但效果受限于背景单一和服装形变处理能力不足。

**技术突破期（2019—2022年）：** 引入三维建模与深度学习结合，代表性工作包括 DiOr（Dressing in Order, 2021），通过循环生成网络实现按顺序穿衣的虚拟试衣，支持多件衣物叠穿。

**扩散模型时代（2022年至今）：** Stable Diffusion、IDM-VTON 等基于扩散模型的方案显著提升了生成质量和细节保真度。Google 的 Gemini 多模态模型具备原生的图像理解与编辑能力，为云端试衣提供了新路径。

### 2.2 现有方案的局限性

| 现有方案 | 局限性 |
|----------|--------|
| 传统 VITON 类模型 | 需本地部署重型 GPU，背景固定，姿势单一 |
| 商业试衣工具 | 功能单一，仅完成"换衣"动作，缺乏风格理解 |
| 扩散模型方案 | 生成质量好但推理慢，提示词依赖人工编写 |
| 现有 LLM+图像组合方案 | 提示词固化，与衣服实际风格脱节 |

### 2.3 本项目的差异化价值

本项目的核心突破在于引入 **"风格DNA驱动"** 机制——先用多模态 LLM 分析衣物的风格基因，再用分析结果动态生成提示词，实现"衣服决定场景"而非"场景随机生成"。这一机制将 AI 试衣从技术工具提升为具有风格理解力的创意助手。

系统采用前后端分离架构，后端基于 FastAPI 异步框架，前端基于 Next.js 14，所有 AI 能力通过 DMXAPI 云端接口调用，无需本地 GPU。

### 3.2 技术选型

| 层次 | 技术选型 | 选择理由 |
|------|----------|----------|
| 视觉分析 | DMXAPI · qwen3.5-flash | 多模态理解能力强，支持图片+文字联合推理，OpenAI 兼容接口易集成 |
| 图像生成 | DMXAPI · gemini-2.5-flash-image | 原生图像编辑能力，可接收多张参考图，输出竖版人像（2:3比例）|
| 后端框架 | FastAPI + asyncio | 原生异步支持，适合 IO 密集型 AI 接口调用，自动文档生成 |
| 前端框架 | Next.js 14 + Tailwind CSS | App Router 支持服务端组件，Tailwind 快速构建赛博朋克 UI |
| 动画库 | Framer Motion | 流畅的入场/出场动画，提升视觉体验 |
| 图片处理 | Pillow | 上传图片规范化、尺寸限制、格式统一 |

### 3.3 核心设计原则

1. **云端化**：去掉本地模型（8GB 显存要求），所有推理通过 API 完成，降低部署门槛
2. **单密钥统一**：两种 AI 服务（视觉分析 + 图像生成）共用一个 `DMXAPI_KEY`
3. **异步降级**：API 失败时返回 mock 图片，保证前端始终有响应
4. **风格驱动**：所有提示词由 LLM 根据衣物 DNA 动态生成，而非固定模板

## 四、关键技术实现

### 4.1 风格 DNA 系统（核心创新）

风格 DNA 系统是本项目最核心的差异化设计，分两个阶段执行：

**第一阶段：DNA 提取**

`analyze_garment_style()` 函数将衣物图片发送给 Qwen VL，提取结构化风格信息。以下为两件风格迥异的服装的 DNA 提取结果对比：

**示例一：街头风格卫衣**

```json
{
  "style_keywords": ["街头", "oversize", "酷感"],
  "color_palette": ["黑色", "米白"],
  "occasions": ["日常", "逛街", "音乐节"],
  "vibe": "都市街头感，慵懒中带叛逆",
  "avoid_scenes": ["正式场合", "商务会议"],
  "pose_suggestions": ["双手插兜，慵懒站姿", "微侧身，随性感", "背靠墙，街头气质"]
}
```

**示例二：修身商务西装**

```json
{
  "style_keywords": ["商务", "正式", "精英感", "修身"],
  "color_palette": ["深藏青", "炭灰"],
  "occasions": ["商务会议", "职场", "正式晚宴", "面试"],
  "vibe": "都市精英气质，干练中透露权威感",
  "avoid_scenes": ["海滩", "运动场", "街头涂鸦墙"],
  "pose_suggestions": ["挺胸直立，双手自然垂落", "单手插袋，侧身四十五度", "俯身整理袖扣，强调细节"]
}
```

两件服装经 Qwen VL 分析后产生了截然不同的 DNA 输出：卫衣的 `avoid_scenes` 明确排除正式场合，而西装则排除休闲街头场景；`pose_suggestions` 也分别对应慵懒随性与挺拔干练两种完全相反的姿势语言。这一差异将在后续阶段直接驱动场景生成的方向。

**第二阶段：DNA 驱动提示词**

提取的 DNA 被注入 `build_plan_prompt()`，驱动 Qwen 生成与服装气质高度匹配的多方案计划：

- 一件西装 → 生成挺拔站姿、高楼大堂、硬光阴影等方案
- 一件卫衣 → 生成慵懒倚墙、街头涂鸦墙、散射自然光等方案

**效果对比：**

| 维度 | 固定列表方案（旧方式）| 风格DNA驱动（新方式）|
|------|---------------------|---------------------|
| 场景来源 | 从预设列表随机选取 | LLM 根据服装气质自由创作 |
| 场景相关性 | 与服装风格无关 | 与服装气质高度匹配 |
| 提示词质量 | 通用、重复 | 个性化、精准 |

### 4.2 多模态视觉分析管线

系统调用链路如下：

```
用户上传图片
    → images.py 验证/压缩/转 JPEG
    → analyze_garment_style()  ← qwen3.5-flash API
    → build_qwen_plan()        ← qwen3.5-flash API（含人物+衣物双图）
    → _normalize_plan()        规范化 + 补全缺失字段
    → apply_mode_constraints() 模式约束覆写
    → _build_variant_prompt()  组装最终图像生成提示词
```

**JSON 三级容错机制：**

Qwen 输出的 JSON 可能包含格式错误（多余文字、markdown 代码块等），系统设计了三级解析策略：

1. 直接 `json.loads()` 解析
2. 提取 `{...}` 块后重新解析
3. 调用 Qwen 文本模型修复 JSON
4. 兜底使用 `build_default_plan()` 固定方案

### 4.3 图像生成与服装锁定

图像生成端调用 DMXAPI Gemini 接口，同时传入人物图和衣物图，由提示词控制生成行为。

**服装类别感知锁定**（解决"只上传上装但裤子也被改"问题）：

```
上装类（衬衫/卫衣/西装等）→ "只替换上半身上装，保持下装/裤子/裙子不变"
下装类（裤子/裙子等）     → "只替换下半身下装，保持上装不变"
其他类别                  → "只替换对应服装，其他保持原图不变"
```

### 4.4 异步任务管理

为解决 AI 接口耗时长（通常 30~90 秒）导致前端超时的问题，系统实现了完整的异步任务流程：

1. 前端 `POST /api/process-advanced-async` → 立即返回 `job_id`
2. 后端在后台线程中运行 Qwen 分析 + 并行 Gemini 生成
3. 前端每 2.5 秒轮询 `GET /jobs/{job_id}` 获取进度
4. 各 variant 图片生成完一张即返回一张，实现"渐进式"展示

### 4.5 并发控制

使用 `asyncio.Semaphore` 全局限制并发 API 调用数量（默认 2），防止单用户请求占用过多 API 配额：

```python
SEM = asyncio.Semaphore(MAX_CONCURRENT)  # 默认 MAX_CONCURRENT=2
async with SEM:
    result = await asyncio.to_thread(call_nanobanana_api, ...)
```

## 七、项目创新点

### 7.1 风格 DNA 驱动的动态提示词生成

**传统方式：** 固定的场景/姿势列表，随机选取，与服装实际风格无关。

**本项目创新：** 首先由多模态 LLM 提取服装的风格基因（style_keywords、vibe、occasions、pose_suggestions），然后将风格 DNA 注入提示词，让 LLM 为每件衣服生成专属的展示方案。同一件卫衣和一件西装，在 StreetShow 中会呈现出完全不同气质的画面。

### 7.2 服装类别感知的精准换衣约束

针对"上传上装但裤子也被改"的常见问题，设计了 `_category_lock()` 函数，根据 Qwen 分析出的服装类别（上装/下装/连衣裙）自动生成针对性的锁定指令，确保 Gemini 只替换用户指定的服装部位。

### 7.3 云端化 + 单密钥架构

将本地重型模型（Qwen3-VL-8B，需 8GB 显存）迁移至云端 API，用一个 `DMXAPI_KEY` 同时驱动视觉分析和图像生成两条链路。部署门槛从"需要 GPU 服务器"降至"任意 Python 环境 + API Key"。

### 7.4 异步任务 + 渐进式展示

通过异步任务队列和前端轮询机制，实现"提交即返回 job_id，生成完一张展示一张"的渐进式用户体验，避免长时间等待的白屏问题。

### 7.5 多物件同时换装

支持一次上传多件衣物（上装+下装，或多件叠穿），AI 理解每件衣物的角色并生成完整穿搭效果，超越现有工具的单件换装限制。

## 十一、总结与展望

### 11.1 项目总结

本项目成功研发了 StreetShow AI 虚拟试衣与穿搭顾问平台，实现了以下核心成果：

1. **完整的云端 AI 管线**：从上传图片到生成专业试衣图，全链路自动化
2. **风格DNA驱动系统**：业界首创以服装气质驱动场景/姿势/光影的动态提示词生成
3. **四种差异化功能模式**：满足标准试衣、多场景展示、姿势变换、多物件上身等不同需求
4. **结构化 AI 建议**：分块展示人物分析、服装解析、搭配建议和风格 DNA
5. **完整工程化实现**：异步任务、历史记录、并发控制、失效降级等生产级特性

系统已实现申报书中规划的全部四个技术模块，并在原规划基础上新增了风格 DNA 系统和 Multi-Fit 多物件上身功能，总体完成度超出预期。

### 11.2 不足与改进方向

1. **图像一致性**：不同 variant 之间的人物身份一致性有时不稳定，后续可探索 ControlNet 或人脸锁定技术
2. **生成耗时**：单次完整生成约 45~90 秒，可引入 WebSocket 实时推送替代轮询
3. **个性化数据积累**：目前历史记录仅存储在本地，未来可接入用户系统实现云端同步和个性化推荐
4. **商业场景适配**：可增加商品图直接替换功能，对接电商平台 SKU 数据

### 11.3 应用前景

StreetShow 系统的潜在应用场景包括：
- **服饰电商商家**：快速生成商品展示图，替代高成本模特拍摄
- **内容创作者/博主**：快速生成多场景穿搭内容
- **消费者购物决策**：上传自己照片查看上身效果，降低退货率
- **服装设计师**：快速验证设计效果和搭配方案

### 附录 B：核心接口文档

**POST /api/process-advanced-async**

```
请求参数：
  person_image   : File   (JPEG/PNG, ≤10MB)
  garment_image  : File   (JPEG/PNG, ≤10MB)
  k_variants     : int    (1~8, 默认4)
  mode           : string (lookbook|pose|multi, 默认lookbook)
  user_prompt    : string (可选，用户偏好描述)

响应：
  job_id   : string  (任务UUID)
  status   : string  (running|completed|error)
  advice   : object  {person, garment, styling, quality}
  plan     : object  {person, garment, style_dna, variants[]}
  results  : array   [{variant_id, title, image_url, prompt_used}]
  meta     : object  {qwen_ms, nano_ms, mode, k_variants}
```

**POST /api/process-multi**

```
请求参数：
  person_image   : File   (人物图)
  garment_images : File[] (衣物图列表，最多3张)
  user_prompt    : string (可选)

响应：
  image_url       : string  (生成结果图URL)
  style_dna_list  : array   (每件衣物的DNA分析)
  garment_count   : int
  meta            : object
```
